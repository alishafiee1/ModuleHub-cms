# رفتار — مدیریت کارت پوشه و دیزاین استاتیک

> **مسیر:** `docs/change/folder-card-management/behavior.md`  
> **برای چیه:** بدون کد بفهمی **کاربر و ادمین روی کارت پوشه و قالب استاتیک چی می‌بینند و چی می‌زنند**.  
> **فنی:** [design.md](./design.md) · **چرا:** [proposal.md](./proposal.md) · **مرجع کلی سایت:** [ui-behavior.md](../../ui-behavior.md) §۱.۱ و §۲.۵

---

## یک جمله

پوشه مثل ماژول **چرخ‌دنده** و **یک خط توضیح** دارد؛ ادمین از همان بوم اسم را عوض می‌کند، جابه‌جا می‌کند یا حذف می‌کند. قالب **استاتیک** HTML داخل ماژول جای آیکون/عنوان را می‌گیرد و روی بوم **قفل** است — نه drag، نه resize.

---

## کی چی می‌بیند؟

| نقش | کارت پوشه | کارت ماژول (مرتبط) |
|-----|-----------|---------------------|
| **بازدیدکننده** | آیکون پوشه، عنوان، توضیح کوتاه (اگر باشد) — **بدون** ⚙ | مثل قبل + توضیح کارت یا یادداشت نسخه |
| **Super Admin** | همان + ⚙ روی آیکون | همان + ⚙ |
| **Super Admin در «ویرایش چیدمان»** | ⚙ **پنهان** — فقط drag/resize (مگر کارت استاتیک: بدون دستگیره) | ⚙ پنهان |

- توضیح زیر عنوان: حداکثر **۲ خط**؛ اگر خالی باشد آن خط اصلاً نیست.
- نشانگر وضعیت (دایره running/stopped): **فقط ماژول** — روی پوشه نیست.

---

## کلیک روی کارت پوشه

| نقطه کلیک | نتیجه |
|-----------|--------|
| **بدنهٔ کارت** (غیر از ⚙) | ورود به پوشه — `/?folder=...` — breadcrumb به‌روز |
| **⚙** | منوی مدیریت پوشه — **داخل پوشه نمی‌روی** |
| **در حالت ویرایش چیدمان** | **دستهٔ دوخط** بالای کارت → drag/resize روی گرید یا انتقال بین پوشه‌ها؛ **کلیک بدنهٔ پوشه** → ورود به پوشه؛ کلیک ماژول بی‌اثر؛ ⚙ غیرفعال |

کلیک ⚙ باید `stopPropagation` داشته باشد تا با ورود به پوشه قاطی نشود.

---

## دیالوگ ⚙ — مدیریت پوشه

### باز شدن

1. Super Admin لاگین کرده.
2. روی ⚙ کارت پوشه می‌زند.
3. دیالوگ با عنوان **«مدیریت پوشه: [نام فعلی]»** باز می‌شود — همان شبکهٔ دکمه‌های `gear-actions-grid` مثل ماژول.

**Module Manager** به پوشه دسترسی ندارد — فقط Super Admin.

### گزینه‌ها

| دکمه | کاربر چی می‌بیند / می‌زند |
|------|---------------------------|
| **ویرایش نام و توضیح** | فرم: «نام پوشه» + «توضیح کارت (زیر عنوان)» — حدود ۲۰۰ کاراکتر توصیه — دکمه ذخیره |
| **جابجایی به پوشهٔ دیگر** | لیست درختی پوشه‌ها — مسیر `خانه > … > مقصد` — خود پوشه و زیرپوشه‌هایش **غیرفعال** — انتخاب «خانه» = ریشه |
| **حذف پوشه** | wizard بسته به پر/خالی بودن (بخش بعد) |

### بازخورد

| حالت | کاربر چی می‌بیند |
|------|------------------|
| **در حال ذخیره** | دکمه‌ها موقتاً غیرفعال یا «صبر کن…» |
| **موفق (ویرایش/جابجایی)** | toast سبز کوتاه — دیالوگ بسته — کارت و breadcrumb **بدون رفرش دستی** به‌روز |
| **موفق (حذف)** | کارت ناپدید — اگر داخل همان پوشه بودی → redirect به والد یا خانه |
| **خطای شبکه** | «ذخیره نشد — دوباره تلاش کن» |
| **جابجایی به مقصد نامعتبر** | «نمی‌توانی پوشه را داخل خودش یا زیرپوشه‌اش ببری» |
| **نام خالی** | «نام پوشه را وارد کن» |

## جریان: ویرایش نام و توضیح

1. ⚙ → «ویرایش نام و توضیح»
2. فرم با مقادیر فعلی پر شده
3. ادمین عوض می‌کند → «ذخیره»
4. **موفق:** عنوان روی کارت + breadcrumb اگر همان پوشه باز است
5. **انصراف:** بدون تغییر

**E2E:** E2E-FCM-01

---

## جریان: جابجایی پوشه

### روش ۱ — منوی ⚙ (خارج از edit یا ترجیح کاربر)

1. ⚙ → «جابجایی»
2. درخت پوشه — مقصد را انتخاب می‌کند
3. تأیید
4. **موفق:** کارت در پوشهٔ والد جدید دیده می‌شود — از محل قبلی حذف — slot خالی در مقصد (مثل ساخت پوشهٔ جدید)
5. **انصراف:** بدون تغییر

### روش ۲ — drag در حالت «ویرایش چیدمان» (پوشه یا ماژول)

1. Super Admin → «ویرایش» در نوار چیدمان
2. از **دستهٔ دوخط** بالای کارت (نه بدنه) می‌کشد روی **کارت پوشهٔ مقصد** یا **کارت برگشت** (برای انتقال به والد)
3. **۲ ثانیه** روی مقصد نگه می‌دارد — **پالس زوم کوتاه** روی مقصد — کارت مبدأ کوچک می‌شود
4. موس را رها می‌کند → کارت **در همان‌جا می‌ماند** (برنمی‌گردد) → **دایرهٔ تیک سبز** ظاهر می‌شود
5. **کلیک تیک** → انتقال انجام می‌شود — toast موفقیت
6. **کلیک بیرون تیک** → لغو — کارت به جای قبلی روی گرید برمی‌گردد

**ناوبری در edit mode:** کلیک روی بدنهٔ **پوشه** یا **بازگشت** → ورود؛ کلیک روی **ماژول** → بدون اثر.

**E2E:** E2E-FCM-02 (drag به پوشه)، E2E-FCM-02b (drag به برگشت)

## جریان: حذف پوشه

### پوشه خالی

1. ⚙ → «حذف»
2. «آیا مطمئنی؟» — بله / خیر
3. **موفق:** کارت حذف — toast

### پوشه پر — انتخاب سرنوشت محتوا

ادمین **یکی** را انتخاب می‌کند:

| گزینه | معنی برای کاربر |
|--------|-----------------|
| **فقط وقتی خالی** (پیش‌فرض امن) | اگر چیزی داخلش باشد: «اول محتوا را خالی کن یا جابه‌جا کن» — حذف نمی‌شود |
| **انتقال به والد** | ماژول‌ها و زیرپوشه‌ها می‌مانند یک سطح بالاتر — خود پوشه حذف |
| **انتقال به پوشهٔ دیگر** | tree picker مقصد — همه منتقل — پوشه حذف |
| **حذف همه** (خطر) | زیرپوشه‌ها و ماژول‌ها هم حذف — باید **نام پوشه را تایپ** کند — اگر ماژول running بود هشدار «اول متوقف می‌شود» |

### بعد از حذف وقتی داخل پوشه بودی

- URL `?folder=حذف‌شده` → redirect به والد یا `/` — **نه** صفحهٔ خراب

**E2E:** E2E-FCM-03

## توضیح کارت روی ماژول (یکپارچه)

در ⚙ ماژول → تنظیمات:

- فیلد **«توضیح کارت»** — همان خط زیر عنوان روی بوم
- **«یادداشت نسخه / changelog»** — جدا — برای تاریخچهٔ نسخه

**روی بوم:** اگر توضیح کارت خالی باشد ولی changelog باشد → همان changelog زیر عنوان (سازگاری قدیمی).

در فرم ویرایش باید روشن باشد کدام روی کارت دیده می‌شود.

## دیزاین استاتیک کارت (فاز دوم)

### بازدیدکننده / ادمین (حالت نمایش)

- کارت ماژولی که `static-template` دارد: به‌جای آیکون/عنوان پیش‌فرض، **محتوای iframe** از `card.html` ماژول
- **بدون** انیمیشن یا اسکریپت داخل قالب — فقط HTML/CSS ثابت
- کلیک روی کارت → همان رفتار ماژول (`/modules/...`) — مگر ناحیه‌ای در قالب تعریف نشده باشد

### Super Admin — ویرایش چیدمان

| نوع کارت | drag | resize | ⚙ |
|----------|------|--------|---|
| پیش‌فرض | ✅ | ✅ | مخفی در edit |
| **استاتیک (`layoutLocked`)** | ❌ | ❌ | مخفی در edit |

- قبل از فعال‌سازی قالب استاتیک، ادمین یک‌بار در حالت پیش‌فرض جا/اندازه را تنظیم می‌کند.
- بعد از switch به استاتیک → جای کارت **قفل** — toast اگر بخواهد بکشد: «این کارت قالب ثابت دارد — برای جابجایی موقتاً به حالت پیش‌فرض برگرد»

### چیزی که عمداً نیست

| فکر می‌کنی | واقعیت |
|------------|--------|
| HTML را در تنظیمات سایت می‌چسبانی | **نه** — فقط فایل داخل بستهٔ ماژول |
| قالب استاتیک زنده به‌روز می‌شود | **نه** — realtime در [card-live-customization](../card-live-customization/proposal.md) |
| پوشه هم قالب HTML دارد | **نه** — فقط ماژول |

## چک سریع — آیا درست پیاده شده؟

| سناریو | انتظار |
|--------|--------|
| مهمان روی پوشه | ⚙ نیست |
| ادمین + توضیح پوشه | زیر عنوان، حداکثر ۲ خط |
| ⚙ روی پوشه | منو — نه ورود به پوشه |
| ویرایش چیدمان | ⚙ پوشه پنهان |
| rename پوشه | breadcrumb هم‌نام |
| حذف پر بدون سیاست | پیام «خالی کن» یا wizard |
| حذف cascade | تایپ نام اجباری |
| کارت static-template در edit | بدون دستگیره resize |
| JSON بدون cardDescription | بدون خطا — مثل قبل |

تست: `npm test` · `npm run test:e2e` · E2E-FCM-01..03
