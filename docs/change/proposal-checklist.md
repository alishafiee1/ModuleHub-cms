# چک‌لیست پروپزال‌های ModuleHub CMS

این فایل کنار roadmap.md نگه‌داری میشه.
هدفش اینه که بدونیم هر پروپزال الان چه وضعیتیه، چی باید اصلاح بشه،
و ترتیب اولویت‌بندی برای جلسات بعدی مشخصه.

> **قانون:** در هر درخواست، فقط یه پروپزال رو اصلاح یا می‌سازیم.

### 🔴 اولویت ۱ — `add manifest/proposal.md`
**رودمپ:** فاز ۱، پلن‌های A تا D
**وضعیت فعلی:** موجود — ۹۲۷۵ بایت
**مشکل احتمالی:**
- محتوای کنونی احتمالاً قبل از رودمپ نوشته شده
- باید معیار موفقیت با «AI-Ready بودن راهنما» آپدیت بشه (طبق roadmap پلن ۱-B)
- ساختار module.json جدید (با فیلدهای backend، card، meta) ممکنه اضافه نشده باشه

**کار لازم:**
- [ ] پروپزال رو بخوان، با رودمپ پلن ۱-A مقایسه کن
- [ ] بخش ساختار module.json رو با نسخه رودمپ هماهنگ کن
- [ ] معیار موفقیت: «AI بدون کمک می‌تونه module.json بسازه» اضافه بشه
- [ ] لحن: اگه فنیه، ساده‌تر بنویس
