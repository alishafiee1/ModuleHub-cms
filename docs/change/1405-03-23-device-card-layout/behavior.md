# رفتار — چیدمان کارت per-device (موبایل / تبلت / PC)

> **مسیر:** `docs/change/1405-03-23-device-card-layout/behavior.md`  
> **برای چیه:** همین‌جا بفهمی **کاربر و ادمین چی می‌بینند و چی اتفاق می‌افته** — بدون جزئیات کد.  
> **فنی:** [design.md](./design.md) · **چرا ساختیم:** [proposal.md](./proposal.md) · **مرجع کلی سایت:** [ui-behavior.md](../../ui-behavior.md) §۱.۱ و §۸.۴

---

## یک جمله

سه **چیدمان جدا** برای PC، تبلت و موبایل داریم؛ مرورگر با **عرض پنجره** یکی را انتخاب می‌کند. کارت‌ها **یک‌باره** به همان چیدمان می‌پرند — نه اینکه مثل عکس کوچک/بزرگ شوند.

---

## سه محدوده عرض

| حالت | عرض پنجره (breakpoint) | عرض بوم / گرید | حس کلی |
|------|------------------------|----------------|--------|
| **دسکتاپ (PC)** | **۱۰۲۴px** به بالا | گرید حداکثر **۱۲۰۰px** · shell حداکثر **۱۲۸۰px** · وسط‌چین | بوم کمی از عرض کامل کوچک‌تر؛ دو طرف فضای خالی |
| **تبلت** | **۶۴۱ تا ۱۰۲۳px** | گرید **تمام عرض** قابل‌استفادهٔ بوم | بدون حاشیه کناری؛ بوم کل container را می‌گیرد |
| **موبایل** | **۶۴۰px** به پایین | گرید **تمام عرض** قابل‌استفادهٔ بوم | مثل تبلت، تمام عرض؛ چیدمان `cardGridMobile` |

**سوئیچ layout** فقط با عبور از **۱۰۲۴** یا **۶۴۱** — یک‌باره، نه تدریجی.

---

## حالت دسکتاپ — چی می‌بینی؟

- **بوم کارت** (`#cardCanvas`) و هدر داخل یک **ستون وسط‌چین** هستند — مثل یک کارت بزرگ روی میز، نه کشیده تا لبه مانیتور.
- **حداکثر عرض گرید کارت‌ها:** حدود **۱۲۰۰px**؛ shell کل (هدر + بوم) تا حدود **۱۲۸۰px** — یعنی دو طرف صفحه **فضای خالی** (پس‌زمینه سایت) می‌مونه.
- **عرض مرورگر رو کم می‌کنی (ولی هنوز دسکتاپی):** از حدود ۱۳۰۰px به سمت ۱۰۲۴px می‌ری، ستون وسط **باریک‌تر** می‌شه و به لبه‌های پنجره نزدیک‌تر می‌شه.
- **متن و آیکون داخل کارت:** با `clamp` و `vw` کمی **کوچک‌تر** می‌شن تا جا بشن — این **انعطاف داخل همون layout دسکتاپ** است، نه عوض شدن چیدمان.
- **سلول‌های گرید:** کل عرض قابل‌استفادهٔ بوم را پر می‌کنند (۳۰ ستون از چپ تا راست بوم **قابل drag** هستند).

### وقتی از دسکتاپ خارج می‌شی

به محض اینکه عرض پنجره از **۱۰۲۴** کمتر بشه → **یک‌باره** چیدمان **تبلت** (`cardGridTablet`) لود می‌شه. دیگه layout دسکتاپ نیست؛ کارت‌ها جاشون عوض می‌شه (مثلاً col/span ذخیره‌شده برای تبلت).

---

## حالت تبلت — چی می‌بینی؟

- **بوم حامل کارت‌ها تمام عرض** داخل `.demo-container` را می‌گیرد — **حاشیه خالی کناری** مثل دسکتاپ نداریم؛ فضا محدوده و باید پر بشه.
- **چیدمان:** از `cardGridTablet` (اگر ادمین جدا تنظیم کرده) یا از مشتق یک‌باره از PC.
- **عرض رو باز هم کم می‌کنی (ولی هنوز تبلتی):** مثلاً از ۹۰۰px به ۷۰۰px — **همون layout تبلت** می‌مونه؛ گرید با عرض واقعی بوم **کش می‌آید** و سلول‌ها کل عرض را پر می‌کنند؛ متن/آیکون داخل کارت دوباره کمی کوچک‌تر می‌شن.
- **گوشه‌های بوم:** کل بوم از چپ تا راست برای drag/resize در edit در دسترس است (بعد از فاز audit دیگه ناحیهٔ «مرده» وسط بوم نیست).

### وقتی از تبلت خارج می‌شی

عرض **۶۴۰** یا کمتر → **یک‌باره** چیدمان **موبایل** (`cardGridMobile`).

---

## حالت موبایل — چی می‌بینی؟

- مثل تبلت: **تمام عرض**، بدون حاشیه کناری برای بوم.
- چیدمان از `cardGridMobile` یا مشتق.
- در بازهٔ باریک (مثلاً ۳۹۰ تا ۶۴۰) باز هم **همان layout موبایل** است؛ فقط سلول‌ها و فونت‌ها با عرض جا می‌شن.
- **چرخ‌دنده و دکمه‌های لمسی** در موبایل بزرگ‌تر از دسکتاپ در نظر گرفته شده (راحت‌تر لمس بشن).

---

## چیزی که عمداً نیست (سوءتفاهم رایج)

| فکر می‌کنی | واقعیت |
|------------|--------|
| با کوچک کردن پنجره، کارت‌ها مثل zoom کوچک می‌شن | **نه** — تا وقتی از آستانه رد نشدی، **همون layout** است؛ فقط بوم/فونت جا می‌شن |
| تبلت همیشه ۷۶۸px ثابت وسط صفحه است | **نه (بعد از audit)** — در **نمایش** تبلت، گرید **عرض واقعی بوم** را پر می‌کند؛ ۷۶۸ فقط **مرجع مشتق‌سازی** از PC است |
| سه layout یعنی سه بار zoom روی یک عکس | **نه** — سه مجموعه `col/row/span` جدا در JSON |
| resize روان بین PC و تبلت | **نه** — فقط در **عبور ۱۰۲۴ / ۶۴۱** یک jump به layout دیگر |

---

## بازدیدکننده (بدون ویرایش)

1. سایت باز می‌شه → layout از API می‌آد.
2. عرض پنجره چک می‌شه → یکی از سه breakpoint.
3. برای هر کارت، grid همان breakpoint (یا fallback / مشتق) اعمال می‌شه.
4. پنجره رو resize می‌کنی:
   - **داخل** یک breakpoint → re-layout همان دستگاه (بوم پر می‌شه، فونت‌ها clamp).
   - **عبور آستانه** → یک refresh؛ چیدمان دستگاه دیگر.

### JSON قدیمی (فقط `cardGrid`)

- اولین بار که layout خوانده می‌شه، سرور برای تبلت/موبایل **یک‌بار** از PC مشتق می‌کنه و **ذخیره** می‌کنه.
- Super Admin ممکنه toast ببینه: «چیدمان تبلت/موبایل از روی PC ساخته و ذخیره شد…»
- دفعهٔ بعد دیگه derive تکرار نمی‌شه مگر خودت عوض کنی.

---

## Super Admin — حالت ویرایش

- دکمه **«ویرایش»** → drag، resize، پس‌زمینه کارت، دستگیره ارتفاع بوم.
- **سه دکمه PC / تبلت / موبایل** (فقط در ویرایش):
  - پیش‌فرض: **PC**
  - سوییچ → اول breakpoint قبلی **ذخیره** (flush) → بعد بوم برای دستگاه جدید
  - هر دستگاه **چیدمان جدا** دارد؛ چیزی که روی تبلت می‌کشی روی موبایل نیست مگر خودت یکسان کنی
- **پایان ویرایش** → ذخیره فوری → صفحه **بدون hard refresh** به حالت نمایش برمی‌گرده (`layout-edit-ended` → render دوباره).
- در ویرایش، `renderCards` عمداً skip می‌شه تا drag وسط کار خراب نشه.
- overlap → کارت به slot خالی یا موقعیت قبلی برمی‌گرده + toast زرد.

### ذخیره

- بعد از drag/resize/پس‌زمینه/ارتفاع بوم: **۵۰۰ms** debounce → `PATCH` با فیلد grid **همان breakpoint فعال** (`cardGrid` / `cardGridTablet` / `cardGridMobile`).

### ارتفاع بوم per-device

هر دستگاه **ارتفاع مستقل** دارد: `folderCanvas.gridRows` (PC) · `gridRowsTablet` · `gridRowsMobile` — با زنجیره fallback مثل design §4.3.

**هنگام سوییچ دکمه‌های PC / تبلت / موبایل:**
1. breakpoint قبلی flush و ذخیره می‌شود.
2. ارتفاع بوم از **همان دستگاه هدف** خوانده می‌شود — نه از دستگاهی که داشتی رویش کار می‌کردی.
3. حداقل ارتفاع = جا دادن همهٔ کارت‌های همان breakpoint (`row + rowSpan`)؛ اگر کارت‌ها پایین‌تر از مقدار ذخیره‌شده باشند، بوم **کوتاه نمی‌شود** تا overlap پیش نیاید.

| سناریو | انتظار |
|--------|--------|
| کاربر هنوز per-device تنظیم نکرده | سرور یک‌بار مشتق می‌کند؛ `gridRowsTablet` / `gridRowsMobile` از کارت‌های همان دستگاه |
| سوییچ PC → تبلت در ویرایش | ارتفاع `gridRowsTablet ?? gridRows` + حداقل جا برای `cardGridTablet` |
| کارت جدید ۵×۵ اضافه شد | سرور هر سه breakpoint را در صورت نیاز bump می‌کند |
| ادمین فقط روی موبایل ارتفاع را زیاد کرد | فقط `gridRowsMobile` عوض می‌شود؛ PC/تبلت دست نخورده |
| نمایش عادی (بدون ویرایش) | ارتفاع از viewport breakpoint فعلی resolve می‌شود |

## چک سریع — آیا درست پیاده شده؟

| سناریو | انتظار |
|--------|--------|
| مانیتور عریض (~۱۴۴۰px) | بوم وسط، حدود ۱۲۰۰px گرید، دو طرف خالی |
| عرض ~۹۰۰px | layout **تبلت**، بوم **تمام عرض** container |
| عرض ~۳۹۰px | layout **موبایل** |
| drag به گوشه راست/چپ بوم در تبلت | کارت داخل بوم می‌ماند |
| ویرایش PC + موبایل جدا → resize به ۳۹۰px | چیدمان موبایل دیده شود |
| سوییچ PC → تبلت → موبایل در ویرایش | ارتفاع بوم هر دستگاه درست؛ کارت‌ها overlap ندارند |
| JSON فقط desktop | بعد از load، tablet/mobile در JSON هست |
| پایان ویرایش | بدون Ctrl+F5 درست نمایش داده شود |
