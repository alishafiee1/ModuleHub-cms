# رفتار UI — قالب HTML استاتیک (معلق، غیرفنی)

> **وضعیت:** معلق — پیاده نشد. فقط مرجع ایدهٔ اولیه.

---

## یک جمله

سازندهٔ ماژول با یک فایل HTML ساده ظاهر کارت را می‌ساخت؛ ادمین فقط یک‌بار جا و اندازه می‌چید؛ بعد کارت قفل می‌شد. **این مسیر فعلاً کنار گذاشته شده** تا روش جدید در card-live-customization طراحی شود.

---

## سازندهٔ ماژول چیکار می‌کرد؟

1. فایل `assets/card.html` می‌ساخت — فقط HTML و CSS، **بدون JavaScript**
2. در `module.json` می‌گفت کارت از این فایل بیاید (`static-template`)
3. ZIP ماژول را آپلود می‌کرد

**نه** از تنظیمات سایت HTML نمی‌چسباند — فقط داخل بستهٔ ماژول.

---

## ادمین روی صفحهٔ اصلی

| مرحله | چی می‌دید |
|--------|-----------|
| قبل از قالب استاتیک | کارت عادی — آیکون، عنوان، توضیح — می‌تواند در «ویرایش چیدمان» جا و اندازه بدهد |
| بعد از فعال شدن قالب | به‌جای آیکون/عنوان، **همان HTML** داخل قاب کوچک (iframe) |
| در ویرایش چیدمان | **نمی‌کشد، resize ندارد** — اگر بکشد پیام: «قالب ثابت دارد» |

ترتیب پیشنهادی: اول جا و اندازه را در حالت عادی تنظیم کن، بعد قالب استاتیک فعال شود.

---

## بازدیدکننده

همان HTML ثابت — بدون ⚙. کلیک روی کارت = ورود به ماژول مثل همیشه.

---

## چیزایی که نبود

| فکر می‌کردی | واقعیت |
|------------|--------|
| HTML در CMS بچسبانی | نه — فقط فایل داخل ماژول |
| عدد زنده روی کارت | نه — آن بخش live جدا بود |
| پوشه هم قالب HTML داشت | نه — فقط ماژول |
| با ماوس اندازه عوض کنی | نه — قفل |
